1. Project Overview
Module 14 expands the Rocket Food Delivery React Native mobile application built in Module 13. The goal is two-fold: expand the Customer app with an Account page and notification options, and add a brand-new Courier app. Both apps share a single codebase, organized by role using Expo Router's folder-based navigation.

The project runs on two separate systems that talk to each other:
Frontend — React Native (Expo): the mobile app the user sees and touches
Backend — Spring Boot (Java): the server that stores and serves data

4. File-by-File Breakdown
4.1  app/_layout.tsx — Root Layout
Purpose: This is the outermost wrapper for the entire app. Think of it like the frame of a house — every room (screen) lives inside it.
Contains:
SafeAreaProvider and SafeAreaView — ensures content does not overlap the phone's status bar or home indicator
Stack navigator — registers index, customer, selection, and courier as valid top-level screens
headerShown: false — hides the default navigation header on all screens
Connected to: Everything. Every screen in the app passes through this file first.


4.2  app/index.js — Login Screen
Purpose: The first screen the user sees. Collects email and password, sends them to the backend, and routes the user based on their role(s).
Contains:
Email and password input fields with validation
fetch() call to POST /api/auth — sends credentials to the Spring Boot backend
AsyncStorage.setItem() — saves accessToken, user_id, customer_id, courier_id, and type to the device
Conditional routing logic based on which IDs were returned:
  Both IDs present  →  navigate to /selection
  Customer ID only  →  navigate to /customer/restaurant/list
  Courier ID only   →  navigate to /courier/deliveries
Connected to: Spring Boot backend (/api/auth). Navigates to selection.js, customer/restaurant/list.js, or courier/deliveries.js.

4.3  app/selection.js — Account Selection Page (NEW)
Purpose: Shown only when a user has both a customer and courier account. Lets them choose which role to use for the current session.
Contains:
Rocket Food Delivery logo (no header or footer navigation — intentional per wireframe)
Two TouchableOpacity cards: Customer (person icon) and Courier (car icon)
AsyncStorage.setItem('type', 'customer' or 'courier') — saves the chosen role before navigating
Connected to: Navigates to customer/restaurant/list.js or courier/deliveries.js. Reads nothing from the backend — purely a routing decision screen.

4.4  app/customer/_layout.js — Customer Tab Navigation
Purpose: Defines the bottom tab bar for the Customer app. Acts like a menu bar that stays visible on all customer screens.
Contains:
Three tabs: Restaurants (faUtensils icon), Order History (faClockRotateLeft icon), Account (faUser icon)
Brand colors: dark background (#222126), coral active tab (#DA583B), white inactive tab
Connected to: Wraps customer/restaurant/, customer/history.js, and customer/account.js.

4.5  app/customer/account.js — Customer Account Page (NEW)
Purpose: Allows the customer to view and edit their contact information for their customer account.
Contains:
Primary Email field — read-only (the email used to log in, cannot be changed here)
Customer Email field — editable
Customer Phone field — editable
UPDATE ACCOUNT button — will send changes to POST /api/account/{id} (connected in backend step)
LOG OUT button — clears navigation and returns to login screen
Connected to: Will fetch from GET /api/account/{id}?type=customer and post to POST /api/account/{id}.

4.6  app/customer/history.js — Order History
Purpose: Displays a list of all orders placed by the logged-in customer.
Contains: (existing from Module 13)
Fetches orders from the backend using the customer_id stored in AsyncStorage
Displays order list with a detail modal on tap
Connected to: Spring Boot backend order endpoints.

4.7  app/customer/restaurant/list.js — Restaurant List
Purpose: The Customer app home screen. Displays all available restaurants with rating and price filters.
Contains: (existing from Module 13)
Rating and Price filter dropdowns
Grid of restaurant cards with images, names, and star ratings
Connected to: restaurantService.js which calls the Spring Boot backend.

4.8  app/customer/restaurant/menu.js — Restaurant Menu
Purpose: Displays the menu for a selected restaurant. Allows the customer to add items and place an order.
Contains: (updated in Module 14)
Product list with quantity +/- controls
Order Confirmation Modal with:
  - Order summary (items, quantities, prices, total)
  - By Email checkbox (sendEmail boolean)
  - By Phone checkbox (sendSMS boolean)
  - CONFIRM ORDER button (disabled/'In Progress' while API call is pending)
Connected to: orderService.createOrder() which passes sendEmail and sendSMS to POST /api/orders.

4.9  app/courier/_layout.js — Courier Tab Navigation (NEW)
Purpose: Defines the bottom tab bar for the Courier app. Same visual design as the customer tab bar, different tabs.
Contains:
Two tabs: Deliveries (faClockRotateLeft icon) and Account (faUser icon)
Same brand colors as customer tab bar
Connected to: Wraps courier/deliveries.js and courier/account.js.

4.10  app/courier/deliveries.js — Courier Delivery Page (NEW)
Purpose: The Courier app home screen. Shows all deliveries the courier needs to manage.
Contains:
Deliveries table with columns: Order ID, Address, Status (button), View (icon button)
Status button with color-coded progression:
  PENDING     →  red button   (#c0392b)
  IN PROGRESS →  orange button (#e67e22)
  DELIVERED   →  green button  (#27ae60)  [locked — cannot change]
Tapping the status button advances it to the next stage and will update the database
Tapping the View button opens a Delivery Details Modal showing:
  - Delivery address, restaurant name, order date
  - Product list with quantities and prices
  - Order total
Business Rule: The list shows all PENDING orders with no courier assigned, plus all orders assigned to the logged-in courier.
Connected to: Will fetch from GET /api/couriers/{id}/orders and patch to PATCH /api/orders/{id}.

4.11  app/courier/account.js — Courier Account Page (NEW)
Purpose: Allows the courier to view and edit their contact information for their courier account. Identical structure to the Customer Account page — only the labels and API parameters differ.
Contains:

Courier Email field — editable
Courier Phone field — editable
UPDATE ACCOUNT button
